0. Projektleírás

Az IdőMester egy webes időpontfoglaló rendszer fodrászszalonok számára. A vendégek online választhatnak szolgáltatást (pl. hajvágás, festés, melír), stylistot és elérhető időpontot az üzlet nyitvatartása és a szakemberek beosztása alapján. A rendszer ütközésmentes foglalást biztosít, alap visszaigazolást ad, és áttekinthető naptárnézetet kínál a szalon számára.

1. Követelmények

A projekt főbb céljai és elvárásai a következők:

  • Egyszerű, letisztult foglalási felület a vendégek számára.
  • Ütközésmentes foglalási logika (egy stylist csak egy vendéggel foglalkozhat adott időpontban).
  • Napi/heti naptárnézet a fodrászok és az adminisztrátor számára.
  • Alap szolgáltatáslista időtartammal és árral.
  • Foglalások módosítása vagy lemondása.
  • Adatok lokális tárolása (JSON/SQLite) a prototípus szintjén.
  • Felhasználóbarát felület mobilon és asztali nézetben egyaránt.

A fenti követelmények az MVP-t (Minimum Viable Product) definiálják. A projekt későbbi fázisai opcionálisan tartalmazhatnak extra funkciókat, mint például e-mail értesítések vagy stylist elérhetőségeinek testreszabása.

3. Erőforrások

A projekt megvalósításához szükséges erőforrások:

  • Fejlesztői erőforrás: 1 fő (SoloDev – Kóró Marcell József).
  • Eszközök: Visual Studio Code / PyCharm, GitHub, Git, Python 3.12.
  • Könyvtárak és frameworkök: Flask vagy FastAPI, SQLite, pytest.
  • Verziókezelés: GitHub repository, Git-flow alapú ágkezelés.
  • Dokumentáció: Markdown és HTML alapú dokumentumok a GitHub Pages oldalon.

A fejlesztés teljes egészében egyéni munka, így a kommunikációs és szervezési csatornák is egy fejlesztőhöz kötődnek.

6. Tesztelés

A rendszer hibamentes működése érdekében a következő tesztelési módszereket alkalmazom:

  • Egységtesztek: foglalási logika, időütközések, szolgáltatás időtartamok.
  • Integrációs tesztek: API végpontokon keresztül végzett foglalás.
  • Kézi tesztelés: tipikus felhasználói forgatókönyvek alapján.
  • Használhatósági tesztek: ellenőrzés, hogy a felület mobilon is jól használható-e.

A fő cél: megbízható, stabil és könnyen használható prototípus.

7. Bemutatás és leszállítás

A projekt végső bemutatása során a következőket ismertetem:

  • A rendszer célja és célcsoportja
  • A foglalási folyamat demonstrációja
  • Naptárnézet és stylist kezelés bemutatása
  • A prototípus technikai működésének rövid bemutatása

Leszállítási mód: GitHub repository + GitHub Pages élő demó.
